SUBSCRIPTION FEATURE IMPLEMENTATION - QUICK REFERENCE
=====================================================

DATABASE MIGRATION COMMANDS
---------------------------
To create and apply the migration for subscription features, run these commands in the Package Manager Console:

1. Add Migration:
   Add-Migration AddSubscriptionTables

2. Update Database:
   Update-Database

Or using .NET CLI:
1. dotnet ef migrations add AddSubscriptionTables
2. dotnet ef database update


NEW MODELS CREATED
------------------
1. Subscription - Stores subscription plan details
   - Id, Name, Description, Price, DurationInDays
   - MaxFileUploads, HasUnlimitedAccess, IsActive

2. UserSubscription - Tracks user subscriptions
   - Id, UserId, SubscriptionId, StartDate, EndDate
   - IsActive, FilesUploaded

3. ApplicationUser - Updated with subscription tracking fields
   - SubscriptionStartDate, SubscriptionEndDate
   - FilesUploadedCount, HasActiveSubscription


SUBSCRIPTION PLANS (Seeded Data)
---------------------------------
1. Free Trial
   - Price: FREE (?0)
   - Duration: 4 hours (represented as 1 day in DB)
   - Max Files: 5 files
   - Features: Limited access

2. Premium
   - Price: ?500
   - Duration: 30 days
   - Max Files: Unlimited (0 = unlimited)
   - Features: All features, unlimited access

SERVICES CREATED
----------------
SubscriptionService (ISubscriptionService)
- HasActiveSubscriptionAsync() - Check if user has active subscription
- GetActiveSubscriptionAsync() - Get user's current subscription
- CanUploadFileAsync() - Check if user can upload files
- IncrementFileUploadCountAsync() - Track file uploads
- IsSubscriptionExpiredAsync() - Check subscription expiration
- GetRemainingFileUploadsAsync() - Get remaining file upload quota
